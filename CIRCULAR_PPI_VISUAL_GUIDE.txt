=======

AZIMUTH ANGLE REFERENCE:
=========================

Starting from East (0°) and going counter-clockwise:

    E   =   0° (East)     →  Right side
   30°  =  30°            →  Between E and N
   60°  =  60°            →  Between E and N
    N   =  90° (North)    →  Top
  120°  = 120°            →  Between N and W
  150°  = 150°            →  Between N and W
    W   = 180° (West)     →  Left side
  210°  = 210°            →  Between W and S
  240°  = 240°            →  Between W and S
    S   = 270° (South)    →  Bottom
  300°  = 300°            →  Between S and E
  330°  = 330°            →  Between S and E
    E   = 360° (=0°)      →  Back to East

COORDINATE SYSTEM COMPARISON:
==============================

CARTESIAN (X, Y):
   Point at (10.5 km, 23.2 km)
   X = 10.5 km East
   Y = 23.2 km North

POLAR (Range, Azimuth):
   Same point in polar:
   Range = sqrt(10.5² + 23.2²) = 25.48 km
   Azimuth = arctan2(23.2, 10.5) = 65.7°

CIRCULAR PPI:
   Plots at Cartesian position on circular background
   Tooltip shows BOTH systems:
      "Cartesian: (10.50, 23.20) km"
      "Polar: Range=25.48 km, Az=65.7°"

READING THE DISPLAY:
====================

STEP 1: Identify the range
   - Count circles from center outward
   - Each circle = one range increment
   - Read range label at top (North) of circle

STEP 2: Identify the azimuth
   - Find nearest radial line
   - Read angle label (N, E, S, W, or degrees)
   - Estimate between lines if needed

STEP 3: Use tooltip for exact values
   - Hover over track point
   - Read exact Range and Azimuth
   - Also see Track ID, Time, Annotation

EXAMPLE:
   Visual estimate: "About 25 km at 60°"
   Tooltip confirms: "Range=25.48 km, Az=65.7°"
   Perfect!

FREQUENTLY ASKED QUESTIONS:
===========================

Q: Why is the center at (0,0)?
A: This represents YOUR radar position. All targets are relative to you.

Q: Can I change the number of range rings?
A: Yes! Edit src/plotting.py, line ~195: num_rings = 5

Q: Can I change azimuth line spacing?
A: Yes! Edit src/plotting.py, line ~216: angles_deg = [...]

Q: What if tracks are outside the display?
A: The display auto-scales. All tracks should fit. Zoom out if needed.

Q: Can I switch back to rectangular view?
A: Yes! Use Display Mode dropdown → "Cartesian (X, Y)"

Q: Why green color?
A: Traditional radar displays use green phosphor. It's the classic look!

Q: Can I print or export this display?
A: Right-click on plot → "Export" → Choose format (PNG, SVG, etc.)

================================================================================

TROUBLESHOOTING:
================

PROBLEM: Circles look oval/stretched
SOLUTION: Resize window to be more square-shaped. Aspect ratio is locked.

PROBLEM: Tooltip not showing
SOLUTION: Hover closer to track points (within 0.5 km). Ensure data is loaded.

PROBLEM: No range rings visible
SOLUTION: Load data first. Rings draw automatically when data is plotted.

PROBLEM: Labels overlapping
SOLUTION: Zoom in/out to adjust spacing. Labels scale with view.

PROBLEM: Display is blank
SOLUTION: Check that data file has x, y, trackid columns. Load data again.
